## 一、总体评估

精调后三层（器件 ⊂ 板卡 ⊂ 整机）架构清晰、同源一致（共享 store），且由真实 `.sr` 驱动。
**新建/精调的界面在 PTO 合规度上明显优于历史资产**：token 化配色、圆角词汇表、面型图标、三级按钮、焦点环、
`＋新增`浮窗、panel-shell 卡片语言基本到位（详见合规表）。主要问题集中在 **3 类**：
① 新组件 `ChassisOverview` 的可访问性与字号有零星欠账；② 竖向告警流在「多传感器对象」下的**空间效率与视觉平衡**仍可提升；
③ 精调后的干净面板与**历史 hw-tree 侧栏/右键菜单**并置，后者的旧蓝调残留与越界圆角/字号形成**新旧断层**。

## 二、优化点（按优先级）

### 🔴 P0 · 应修（一致性/可访问性硬伤）

**F1 · `ChassisOverview` 无焦点环（键盘不可见）**
- 证据：`ChassisOverview.vue` `:focus-visible` = 0 处；`.cp-close`（`all: unset`）与 `.cov-row`（hover 态）均无键盘可见焦点。
- 对比：`AlarmConfigView` 有 14 处焦点环、`topology.css` pp-tab/dev-item 有环。整机面板成为唯一「键盘导航黑洞」。
- 优化：为 `.cp-close`、可交互行补 `:focus-visible { box-shadow: 0 0 0 2px var(--focus-ring) }`，与其余面板对齐。

**F2 · 派生「系统状态 System」器件详情「可监控量」为空**
- 证据：`TopologyView.vue:258` 派生器件 `quantities: []` → 器件详情 tab「可监控量」字段渲染空值。
- 优化：`.sr` 派生器件的可监控量应从其 cfgs 反推（如 `离散状态`），或字段为空时隐藏该行（避免空标签）。

### 🟡 P1 · 建议优化（体验/视觉效率）

**F3 · 竖向告警流：多传感器对象下左列留白 + 平衡**
- 现象（实测 System 对象 5 传感器）：监控对象节点为左列垂直居中的小卡，随传感器增多，其上下留白拉大；
  一屏「对象列」信息密度偏低。
- 优化候选：① 对象节点改为**该组顶部的通栏头**（真正竖向：对象在上、其传感器在下），连线走短竖向；
  ② 或保留左置但**顶对齐 + sticky**，长列滚动时对象名常驻；③ 左列宽度按对象名自适应收窄。

**F4 · 整机「各板告警覆盖」表 8/10 行为空（`— —`）**
- 现象：10 块板中仅 ExpBoard_1 有 `.sr`，其余 8 行全 `— —`，表体空重。
- 优化：**已配置置顶 + 未配置折叠/淡化**；或表头给「已配置 N/共 M」摘要，未配置默认收起可展开。

**F5 · 整机统计数字字号越界 PTO 字阶**
- 证据：`ChassisOverview.vue:107` `.stat-n: 18px`；PTO 字阶为 14/16/20/28，无 18。
- 优化：改 `20px`（title-md）或 `--font-size-title-sm 16px`，与字阶统一。

**F6 · 告警工具条上下文与面板头重复**
- 现象：告警 tab 顶部「来自拓扑 ExpBoard_1 · EXU」与面板头「板卡配置 / ‹器件名」信息重叠，窄面板里占一行。
- 优化：窄面板内弱化/去除工具条的板卡重述，仅保留「＋新增」；或把来源并入面板头副标题。

**F7 · 严重度中间档（Major = warning↔danger 55% 混色）辨识度**
- 现象：Major 呈 warning 与 danger 之间的橙红过渡，快速扫视时与 warning（预警）区分度偏弱。
- 优化：引入一档专用中间色 token（如 `--ark-orange-600`），或加非色编码（图标/字重）辅助分级。

### 🟢 P2 · 相邻资产/收尾（非本次告警范围，但并置可见）

**F8 · 历史 hw-tree 侧栏 / 右键菜单 与新面板断层**
- 证据：`TopologyView.vue` 侧栏（约 675–1030 行）仍带**旧蓝调 fallback**（`var(--token, #6b7498/#98a0b8/#c3c9de/#e6e8ef)`）、
  **越界圆角** 4px×5、**sub-11px 字体** 10/10.5px×9；`topology.css:312` 右键菜单 4px 圆角。
- 影响：这些区域与已清洗的 PTO 面板并置，蓝调/小字/小圆角的断层在同屏可见。
- 优化：按阶段一 C1/C8/C9 同法清 fallback、圆角/字号归阶梯（属「收尾对齐」，可安排在告警定稿后统一做）。

**F9 · 竖向连线在密集对象下的观感**
- 现象：对象→多传感器的贝塞尔扇出在长列下线条较多。属可接受，但可选**降权**（更细/更淡）或对超过 N 条时改「竖脊 + 短支」。
