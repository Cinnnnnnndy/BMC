# openUBMC Studio 更新日志与交互说明

> 2026-07-16 · 设计原型（Web Demo）· 在线版：Pages 站点 `release-notes.html`（欢迎页 →「新功能介绍」）

## 更新日志（2026-07-16）

### 底栏 agent 终端（NEW）
- 工作区底部新增 openUBMC agent CLI 终端 dock：`agent <任务>` 一句话派发任务，自动回放 MCP 工具调用轨迹，完成后分屏打开关联功能视图。
- 命令：`agent` / `mcp` / `open` / `views` / `clear` / `exit`；快捷键 `Ctrl+\``；胶囊手柄拖拽调高。
- 词表覆盖 10 个 agent 任务场景、7 个 MCP Server / 26 个工具。

### AI 辅助安装引导（NEW）
- 安装引导 Step 2.2「配置检测」失败横幅新增「让 agent 诊断修复」入口：`setup_verify_all` 六项诊断 → 修复 conan remote / init.py。
- 修复完成后检测状态自动回写向导：Step 2.2 翻绿「已完成」，「下一步」解锁。
- AI 助手新增「开发环境诊断与修复」示例场景（含写操作审批门对话）。

### CSR 拓扑 · 硬件管理器（NEW）
- 左侧面板改为资源管理器式文件树：板卡类别 → 板卡组 → .sr 板卡文件。
- 组行带解析状态色点并与画布联动聚焦；点击文件行切换画布上该组展示的板卡实例；检查汇总与未分类「指派」保留。

### 全局状态栏（UI）
- IDE 底部新增状态栏：BMC 可达数 / SSH 会话 / Docker 编译容器 / 问题统计 ｜ 当前工程 / MCP 工具数 / 终端开关。
- 环境管理页内的底部蓝条移除，连接状态全局仅保留一处。

### 信息架构收纳（UI）
- 出包 / 巡检 / CSR 校验从顶栏下放至功能页 tab 行（editor-title-actions 惯例）。
- 活动栏「···」菜单收纳硬件拓扑、3D 仿真、事件配置（Beta）与四个代码辅助扩展占位页。
- 顶栏换用 openUBMC 官方 logo。

### 修复（FIX）

## 交互说明

### 布局总览
活动栏（功能入口 + 「···」收纳）｜顶栏（品牌 / 当前工程 / 全局动作：终端 · 全屏 · 深浅色 · AI）｜工作区（多 tab 多 pane，拖拽分屏，tab 行右侧为上下文动作）｜底栏终端（执行流）+ 右侧 AI 面板（对话流）｜状态栏（连接与问题，条目可点击直达）。

### 上下文动作（tab 行右侧）
| 激活的功能页 | 动作 | 行为 |
|---|---|---|
| BMC 环境管理 | 巡检 | `agent 巡检在线 BMC` |
| 拓扑 / CSR 拓扑 / 事件 / 传感器 | 校验 | `agent 校验当前 CSR` |
| CSR 编辑类视图（已加载工程） | 出包 | 下载/保存 CSR，无修改置灰 |

### agent 终端命令
| 命令 | 说明 |
|---|---|
| `agent <任务描述>` | 匹配任务场景，回放 MCP 工具轨迹，完成后分屏打开关联视图 |
| `mcp` | 列出 7 个 MCP Server / 26 个工具 |
| `open <视图>` | env / topo / event / sensor / smc / expr / explorer / setup / sim / 3d / cooling / welcome / ai |
| `views` | 列出视图别名 |
| `clear` / `exit` | 清屏 / 关闭终端 |

### AI 辅助安装引导路径
安装引导 → 登录 → 下一步 → 配置检测（失败）→ 让 agent 诊断修复 → 终端诊断 6 项、修复 2 项 → Step 2.2 翻绿、下一步解锁。
账号授权类操作（登录官网、GitCode 公钥上传）仍需本人完成；写操作在 AI 面板对话版中有审批门。

### 硬件管理器（CSR 拓扑左栏）
类别 → 板卡组（状态色点：绿已解析/橙多匹配/黄待选/红缺失/灰未分类，点击联动画布）→ 文件行（`PartNumber_…SN尾号.sr`，悬停看全名，点击切换画布实例）；未分类可「指派」到空槽位。

### 状态栏条目
BMC 3/5 可达（→环境管理）｜SSH 1（→环境管理）｜Docker 2/3（→巡检任务）｜⊗0 ⚠2（→输出面板）｜MCP 7（→终端工具清单）｜终端（开关）。
