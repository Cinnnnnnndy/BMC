# 数据模型 — openUBMC Studio 核心数据结构

> 本文档说明系统中所有核心数据结构的字段含义和设计意图。  
> 源文件：`csr-topology-editor/web-app/src/data/hardwareTopologyData.ts`  
> 使用方式：作为 AI 生成代码或解释数据逻辑时的精确参考。

---

## 一、板卡层：HwBoard

系统中对一块物理板卡的完整数据表示。

```typescript
export interface HwBoard {
  uid: string;              // 唯一标识符，20位数字字符串（来自 CSR .sr 文件）
  name: string;             // 板卡型号，格式 "BC83XXXX"，如 "BC83AMDA"
  type: string;             // 板卡类型：PSR | BCU | CLU | IEU | EXU | SEU | NICCard | BMC
  desc: string;             // 人类可读描述，如 "CpuBoard"、"NVMe*24+SAS*12"
  topoNodes: TopoNode[];    // I²C 拓扑树节点列表（树形结构，用 parent 字段组织）
  businessConnectors: BusinessConn[];  // 业务层接口列表（PCIe、OCP 等）
  soft: SoftOverlay;        // 软件叠加层数据
}
```

**关键约定**：
- 同一 `type` 的多块板卡在分组视图中合并显示（如 IEU 有多个型号，同归 IEU 分组）
- `uid` 是全局唯一的，`type` 是业务分类，二者不可混用
- `name` 总是以 "BC83" 开头（CSR 平台板卡命名规范）

## 二、拓扑层：TopoNode

I²C 树上的单个节点，代表一个物理芯片、总线或连接器。

```typescript
export interface TopoNode {
  id: string;               // 节点唯一 ID，人可读格式如 "I2c_1"、"Smc_ExpBoardSMC"
  parent: string | null;    // 父节点 id；null 表示根节点（仅 Anchor）
  type: string;             // 节点类型（见下方类型表）
  address?: number;         // I²C 设备地址（十进制），总线节点无此字段
  channel?: number;         // MUX 通道号（0-7），非 MUX 子节点无此字段
}
```

### TopoNode 类型表

| type | 含义 | 典型 id 格式 | 地址范围 |
|---|---|---|---|
| `anchor` | 虚拟根节点，树的起点 | `"Anchor"` | 无 |
| `i2c_bus` | I²C 总线 | `"I2c_1"`, `"I2c_2"` | 无 |
| `i2c_mux` | I²C 多路复用器 | `"I2cMux_1"` | 0x70–0x77 |
| `smc` | 管理微控制器 | `"Smc_ExpBoardSMC"` | **固定 0x60** |
| `eeprom` | FRU 存储芯片 | `"Eeprom_BCU"` | 0xAE（174） |
| `connector` | 物理连接器 | `"Connector_Memory"` | 无 |
| `cpld` | 可编程逻辑器件 | `"Cpld_1"` | 无 |
| `jtag` | JTAG 调试接口 | `"JtagOverLocalBus_1"` | 无 |
| `chip` | 通用芯片 | `"Chip_PowerCtrl"` | 各异 |

### 树形结构规则

```
Anchor (parent=null)
├── I2c_1 (type=i2c_bus)
│   ├── Smc_CpuBrdSMC (type=smc, addr=0x60)
│   ├── Eeprom_BCU (type=eeprom, addr=0xAE)
│   ├── Connector_Memory (type=connector)
│   └── I2cMux_1 (type=i2c_mux, addr=0x72)
│       ├── Chip_Temp1 (channel=0, type=chip, addr=0x48)
│       └── Chip_Temp2 (channel=1, type=chip, addr=0x48)  ← 同地址，不同 channel
├── I2c_2 (type=i2c_bus)
│   └── Smc_ExpBoardSMC (type=smc, addr=0x60)  ← 每块板必有
└── JtagOverLocalBus_1 (type=jtag)
```

**I2c_2 总线**：每块板卡固定存在，包含 `Smc_ExpBoardSMC`（addr=0x60），这是 PSR 管理该板卡的唯一入口。

## 三、业务层：BusinessConn

描述板卡对外的业务接口（PCIe 槽、OCP 接口等），与 I²C 拓扑是平行的两个视角。

```typescript
export interface BusinessConn {
  id: string;               // 接口唯一 ID
  name: string;             // 接口名称，如 "Hisport_0"、"CEM_0"
  direction: string;        // "Upstream"（上行，本板插入上游）| "Downstream"（下行，供下游插入）
  linkWidth: string;        // PCIe 链路宽度，如 "x16"、"x8"、"x4"
  maxRate: string;          // 最大速率，如 "Gen4"、"Gen5"
  connType: string;         // 连接器类型：Hisport | CEM | OCP | UBC | UBCDD | ...
  slot?: number;            // 插槽编号（Downstream 接口）
  refConnector: string;     // 桥接字段：对应 TopoNode 中某个 connector 节点的 id
  upstreamNames: string[];  // 上行目标的 connector id 列表（描述插入哪个槽）
}
```

**方向语义**：
- `Upstream`：本板卡对外输出的上行接口（本板"插入"上游板卡的 Downstream 槽）
- `Downstream`：本板卡提供给下游设备插入的插槽

**refConnector 的作用**：将业务层（接口类型、速率）与拓扑层（I²C 芯片节点）关联，是两个数据视角的桥梁。

## 四、软件层：SoftOverlay

软件视角的叠加数据，在硬件拓扑上额外显示软件行为信息。

```typescript
export interface SoftOverlay {
  fruData: Array<{
    id: string;       // FRU 设备 ID
    chip: string;     // 绑定的 TopoNode id（通常是 eeprom 节点）
  }>;
  components: Array<{
    id: string;       // 软件组件 ID
    type: number;     // 组件类型编号
  }>;
  scannerChips: Record<string, string[]>;  // { scannerName: [topoNodeId, ...] }
  eventCount: number;    // 该板卡定义的软件事件数量
  sensorCount: number;   // 该板卡定义的传感器数量
}
```

**开启方式**：UI 中 `showSoft: true`（CVS 状态），在 MiniBusDiagram 芯片节点上叠加显示 Scanner 引用数和 FRU 绑定标记。

## 五、画布状态：CVS（Canvas View State）

记录当前可视化视口的完整状态，可序列化保存和恢复。

```typescript
interface CVS {
  offset: { x: number; y: number };  // 画布偏移量（px），应用 scale 变换前
  scale: number;                       // 缩放比例，范围 0.15–2.5
  filterType: string | null;           // 当前筛选的板卡类型，null=不筛选
  showSoft: boolean;                   // 是否显示软件叠加层
  showI2C: boolean;                    // 是否显示 I²C 管理总线连线
  showPCIe: boolean;                   // 是否显示 PCIe 连线（默认 true）
  highlightUids: string[];             // 高亮的板卡 uid 列表，空=全部正常显示
  selectedBoardUid: string | null;     // 当前选中的板卡 uid
}
```

**坐标变换公式**：
```
屏幕坐标 = 画布坐标 × scale + offset
画布坐标 = (屏幕坐标 - offset) / scale
```

## 六、板间连线：BOARD_CONNECTIONS

定义板卡类型之间的连接关系，驱动 SVG 连线渲染。

```typescript
const BOARD_CONNECTIONS = [
  {
    fromType: string;    // 源板卡类型
    toType: string;      // 目标板卡类型
    label: string;       // 连线的语义标签（hover tooltip 显示）
    color: string;       // 连线颜色（十六进制）
    style: 'solid' | 'dashed';  // 实线=PCIe，虚线=I²C
    sharedI2C: boolean;  // true=Hisport双轨，false=单轨连线
  }
]
```

**完整连接表**：

| fromType | toType | 类型 | 标签 | 特殊 |
|---|---|---|---|---|
| PSR | BCU | I²C 虚线 | I²C 管理总线 | — |
| PSR | CLU | I²C 虚线 | I²C 管理总线 | — |
| PSR | EXU | I²C 虚线 | I²C 管理总线 | — |
| PSR | SEU | I²C 虚线 | I²C 管理总线 | — |
| BCU | IEU | PCIe 实线 | Hisport (PCIe + I²C侧带) | **sharedI2C=true** |
| BCU | EXU | PCIe 实线 | PCIe Hisport↔OCP | — |
| IEU | SEU | PCIe 实线 | PCIe CEM↔UBC | — |
| IEU | NICCard | PCIe 实线 | PCIe CEM | — |
| EXU | NICCard | PCIe 实线 | OCP↔PCIe | — |
| EXU | SEU | PCIe 实线 | PCIe CEM↔UBC | — |

## 七、布局相关常量

```typescript
const BOARD_W = 260;          // 展开视图板卡卡片宽度（px）
const BOARD_H = 180;          // 展开视图板卡卡片最小高度（px）
const GROUP_W = 250;          // 分组视图卡片宽度（px）
const GROUP_H = 240;          // 分组视图卡片默认高度（px）
```

**高度估算函数** `estimateCardH(nodes: TopoNode[])`:
- 取前 5 条 Anchor 直连总线
- 每条总线：22px（总线行高）+ ceil(min(chipCount, 12) / 5) × 46px（芯片行）
- 最小返回 210px
