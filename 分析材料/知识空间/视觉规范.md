## 一、颜色系统

### 1.1 背景层级

| 用途 | 色值 | 描述 |
|---|---|---|
| 页面/应用背景 | `#060a0f` | 最深层，几乎纯黑 |
| 画布背景 | `#060a0f` | 同页面背景，无分隔感 |
| 卡片背景 | `#0b1118` | 比画布稍亮 |
| 选中卡片背景 | `#0f1f35` | 带蓝调的深色 |
| 面板/侧栏背景 | `#0b1118` 或 `#060a0f` | 与卡片同层或更深 |
| 分组标题区背景 | `rgba(0,0,0,0.3)` | 在卡片内的深色标题 |

## 二、卡片结构规范

### 2.1 展开视图 BoardNode 卡片

```
┌─────────────────────────────────────────┐  ← border: 1px solid {typeColor}44（未选中）
│ 标题行: 类型徽章 | 板卡名称 | 操作按钮  │    border: 1px solid {typeColor}（选中）
│ ──────────────────────────────────────  │  ← 分隔线
│ 描述文字（可选）                         │
│                                          │
│ MiniBusDiagram                          │  ← I²C 总线小图
│  ┌─────────────────────────────────┐    │
│  │ I2c_1 总线行                    │    │
│  │ [SMC][EEPROM][CONN][CONN]       │    │
│  │ I2c_2 总线行                    │    │
│  │ [SMC]                           │    │
│  └─────────────────────────────────┘    │
│                                          │
│ BusinessConnector 列表（若有）           │
└─────────────────────────────────────────┘
```

- **卡片宽度**：`BOARD_W = 260px`
- **最小高度**：`BOARD_H = 180px`，实际由内容撑开
- **圆角**：`borderRadius: 8px`
- **阴影（未选中）**：`0 2px 8px rgba(0,0,0,0.4)`
- **阴影（选中）**：`0 0 16px {typeColor}55`
- **Cursor**：可拖拽时 `grab`，选中操作时 `pointer`

### 2.2 分组视图 GroupCard 卡片

与 BoardNode 结构类似，但有以下差异：
- 宽度：`GROUP_W = 250px`
- 标题区：类型徽章 + 板卡名称（下拉）+ 数量角标（多板时）
- 内部显示：当前激活板卡的 MiniBusDiagram

### 2.3 标题区徽章

```
┌────────────────────────────────┐
│ [BCU] BC83AMDA ▾  ×3          │  ← [类型徽章] 名称▾（下拉）  ×N（多板时）
└────────────────────────────────┘
```
- 类型徽章：`{typeColor}22` 背景，`{typeColor}` 文字，`borderRadius: 3px`，`padding: 1px 5px`，`fontSize: 9px`
- 名称：`fontSize: 11px`，`fontWeight: 600`
- 下拉箭头 ▾：`fontSize: 7px`，`color: #64748b`，紧跟名称后
- 数量角标 ×N：`fontSize: 9px`，`color: {typeColor}99`，右侧对齐

## 三、节点芯片图标规范

### 3.1 普通芯片图标

```
┌──────────┐
│ 节点类型  │  高度: 16px
│ 缩写名称  │  宽度: 22px（connector 为 24px）
└──────────┘
```

- 普通节点：`22×16px`，`borderRadius: 3px`，背景色 = `NODE_COLOR[type] + '33'`（20%透明度），边框 `1px solid NODE_COLOR[type]`
- connector 节点：`24×14px`，`borderRadius: 7px`（胶囊形）

### 3.2 SMC 节点

SMC 是板卡的管理核心，在图标右上角有一个绿色小圆点：
```
[SMC] ← 右上角绿点 (width:5px, height:5px, borderRadius:50%, background:#22c55e)
```

### 3.3 软件叠加标记

当 `showSoft=true` 时，在芯片图标右上角叠加：
- **Scanner 引用**：`🟢 N`（N 为引用该芯片的 Scanner 数）
- **FRU 绑定**：`🟡 FRU`

## 四、连线渲染规范

### 4.1 bezier 曲线参数

连线从源卡片右侧中点出发，到目标卡片左侧中点结束：
```
path d = `M ${x1} ${y1} C ${cx1} ${y1}, ${cx2} ${y2}, ${x2} ${y2}`
```
控制点 x：`cx1 = x1 + (x2-x1)*0.5`，`cx2 = x1 + (x2-x1)*0.5`

### 4.2 连线宽度

| 状态 | strokeWidth |
|---|---|
| 默认 | 1.5 |
| hover | 2.5 |
| I²C 虚线 | 1（dasharray: 4 3） |

### 4.3 Hisport 双轨渲染（sharedI2C=true）

同一物理连接渲染两条平行线：
- 上轨（PCIe）：`color` 色，偏移 -2px
- 下轨（I²C侧带）：`#0ea5e9`（I²C蓝），偏移 +2px
- hover 时两轨同时高亮

### 4.4 SVG 图层管理

```
z-index 层叠关系：
画布背景 (z:0)
板卡卡片 (position:absolute, z:10)
连线 SVG (position:absolute, z:20)
  ├── SVG 容器：pointerEvents: none（穿透到卡片）
  └── <g> 元素：pointerEvents: all（保留连线交互）
工具栏/面板 (z:30+)
```

## 五、交互规范

### 5.1 拖拽

- 触发：在卡片任意位置按住鼠标（排除 `button/input/select/textarea`）
- 拖拽时：`cursor: grabbing`
- 区分拖拽和点击：`draggedRef.current`，mousemove 设为 true，mousedown 重置为 false，onClick 判断后决定是否触发 `onSelect`

### 5.2 缩放

- 触发：滚轮事件
- 范围：`0.15×` 到 `2.5×`
- 锚点：鼠标当前位置（保持鼠标下方内容不动）

### 5.3 高亮/选中

- **高亮**（搜索/筛选触发）：非高亮卡片 `opacity: 0.2`
- **选中**（点击触发）：边框加亮 `1px solid {typeColor}`，加发光阴影
- 可同时有高亮状态和选中状态

### 5.4 Tooltip

连线 hover 时显示 tooltip，包含：连线标签（如 "Hisport (PCIe + I²C侧带)"）+ 两端板卡类型名称。

## 六、排版规范

| 用途 | fontSize | fontWeight | color |
|---|---|---|---|
| 应用标题 | 13px | 700 | #e2e8f0 |
| 卡片板卡名 | 11px | 600 | #e2e8f0 |
| 类型徽章 | 9px | 500 | {typeColor} |
| 总线标签 | 9px | 500 | #94a3b8 |
| 芯片图标文字 | 7–8px | 400 | NODE_COLOR[type] |
| 说明文字/描述 | 10px | 400 | #64748b |
| 角标/次要 | 9px | 400 | {typeColor}99 |

**字体**：系统字体栈，不指定特定字体，优先 monospace 风格的系统字体（适合工程工具）。
